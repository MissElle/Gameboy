Responsive Web Design

and the many problems it solves

Also, there's a form...

Responsive web design is the solution to how the majority of users both view and interact with the web. Until recently, there was a standard viewport used by a user: the monitor. True the aspect ratio of the monitor would change from time to time, but it was predictable. With the advent of the mobile web, viewports now widely vary in size, ratio, and even input from the user.

By creating a site that can interact with and respond to the viewport, a developer need not worry about what platform a user is using. In many ways, responsive web design is a medium used by a developer to maximize usability, function, and cross-platform stylization.

The developer does this by using relative sizing. For example, the developer can code a web page to be sectioned into thirds no matter how large or small the viewport is. Similarly, the sizing of text, images, and sectioning can also be coded to change with the sizing of the viewport in real time. This creates a seamless, smooth experience. Ultimately, is that not what developers want for their users?
